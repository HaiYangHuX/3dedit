# 资源版本字段简化设计

## 背景

模型与素材库此前引入了版本历史、版本状态、版本说明和版本唯一性校验。当前产品阶段只需要在资源上保存一个可编辑的版本文本，历史版本会让上传替换流程变复杂，也会让详情页展示偏离当前产品定位。

## 目标

- `Asset.version` 作为资源普通元数据字段保留，允许业务自定义文本，最多 40 个字符。
- 详情页只展示当前版本文本，不返回、不渲染版本历史。
- 编辑资源可以直接修改版本文本。
- 模型文件修改通过“替换模型文件”完成：沿用同一个 `assetId`、更新当前源文件和解析结果，不产生版本记录。
- 上传替换未传版本时保留原资源版本；显式传入时才更新该字段。

## 非目标

本阶段不实现版本递增、版本语义化格式校验、版本唯一性、版本状态机、版本说明、版本时间线、版本回滚或版本对比。

## 数据与 API

- Prisma 删除 `AssetVersion` 模型、`Asset.versionNotes`、`UploadSession.version` 和 `UploadSession.versionNotes`。
- 上传会话仅保留元数据快照，以保证上传取消或失败时不会覆盖当前资源信息；快照中的 `version` 只是待写回的普通字符串。
- `AssetService` 列表和详情 DTO 只映射 `version` 字段及当前文件信息。
- Worker 只更新 Asset 的解析状态、元数据、activeFile 和 sourceHash，不同步任何版本表状态。
- 新增迁移 `0007_remove_asset_version_logic` 清理历史表和旧字段；历史迁移保留以保证全新数据库可顺序部署。

## 前端交互

- 添加/编辑弹窗保留一个普通版本输入框，不显示版本说明，不做 semver 校验，不自动递增。
- 详情中的“上传新版本”改为“替换模型文件”，使用同一上传弹窗和 `assetId`。
- 详情侧栏移除版本时间线，当前版本只在标题和基础信息中展示。
- 资源信息编辑保存时允许用户清空版本文本。

## 验收标准

1. API 合约可接受 `release-candidate` 等非 semver 文本，并拒绝超过 40 字符的值。
2. 资源详情 JSON 不包含 `versions` 或 `versionNotes`。
3. 完成替换上传后仅存在一个 Asset，`Asset.version` 按输入或原值保存，数据库中不创建 `AssetVersion`。
4. 替换模型后详情预览、源文件下载地址和解析状态指向新文件。
5. API、Worker、前端类型检查和测试全部通过，Prisma 迁移状态为 up to date。
